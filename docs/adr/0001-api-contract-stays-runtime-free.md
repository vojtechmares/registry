# The API contract package stays runtime-free

`@registry/api-contract` is imported type-only by both the Worker and the web app. Moving the valibot schemas into it - to get "one declaration" for request bodies - was considered and rejected: the request schemas validate with functions from seven feature packages (`regex`'s NFA compiler, `semver`, `cron`, `oci`, `projects`, `notifications`, plus email checks), so a runtime contract package would turn the leaf of the dependency graph into a hub and drag those packages into the web app's graph. Instead, request-input types are hand-written in the contract and pinned to the valibot schemas by bidirectional compile-time assignability guards (the `Describes<>` block in `apps/registry/src/api/schemas.ts`); response types stay guarded one-directionally. Every request and response schema must have a contract type and a guard entry - the shapes that escaped the guard were exactly the ones that drifted.
